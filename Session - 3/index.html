Introduction to JavaScript API

JavaScript APIs (Application Programming Interfaces) are sets of tools and protocols that allow developers to interact with web browsers and other services. They provide pre-defined functions and methods to perform common tasks, such as manipulating the DOM, handling events, making network requests, and working with multimedia.

4 - async/await

The async/await syntax in JavaScript is used to handle asynchronous operations in a more readable and manageable way. It allows you to write asynchronous code that looks and behaves like synchronous code, making it easier to understand and maintain.

Key points:

  • `async` before function means "this function has async operations"
  • `await` means "wait for this to finish before moving on"
  • `fetch()` makes the HTTP request
  • `json()` converts the response to JavaScript object

5 - Error Handling

Error handling in JavaScript APIs is crucial for ensuring that applications can gracefully handle unexpected situations, such as network failures, invalid responses, or other issues that may arise during API interactions. Proper error handling helps improve the user experience and maintain application stability.

  • `try { }` - Try to run this code
  • `catch (error) { }` - If anything fails, run this code instead
  • `throw new Error()` - Manually create an error
